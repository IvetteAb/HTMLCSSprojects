Introduction

HTML stands for Hypertext Markup Language, which tells the browser where the content belongs on the webpage. There are various elements you can use to structure your content and make it behave in a certain way.

Tags

Example:

HTML code: <p>This is a paragraph.</p>

Outcome: This is a paragraph.

There are two types of tags:

  • opening tag: which is used at the start of paragraph.
  • closing tag: This is used at the end of the content and includes a forward slash before the name of the element.

Format: Opening tag, content then closing tag.

Note: Not every element has a closing tag. You may encounter an empty (or void) element. Single tags are generaly used for embedding items in the document, such as images.


Nesting elements

A nested element (also referred to as a child element) is when a HTML element contains additional elements, such as attributes, tags etc.

For example:

HTML code: My dog is <strong>excited</strong> to go to the park today.

Outcome: My dog is excited to go to the park today.

There are two uses of bold:

  • <strong> & </strong> = to show importance; seriousness; urgency e.g. section headings
  • <bold> & </bold> = its main purpose is to jump out to the reader, it is generic; neutral; has no meaning or alternative voice.

There are two uses of italics:

  • <em> & </em> = emphasis italics (verbally emphasised)
  • <i> & </i> = visual-only italics (generic; neutral; it has no meaning)


Block and inline elements

Elements can be sorted into block-level and inline elements.

  • Block elements creates a new block on a page and wiill take up the maximum width available. The browser will also add a margin (space) before and after the element. Block elements cannot be nested within an inline element, but can be nested in another block element.

    • Some common block elements would be <p> and <div>.


  • Inline elements are used within block elements (so they do not begin a new line on the document) and only takes up the necessary width of a page.

    • Some examples are <a> (for creating hyperlinks) and <i>.

E.g.

HTML code:<p>A</p><p>B</p><p>C</p> <i>D</i><i>E</i><i>F</i>

Outcome:

A

B

C

DEF

First we have <p> which is a block element, so each letter begins on a new line with spacing between each letter. On the other hand, <i> is an inline element, so the first few letters are on the same line which has no spaces between them.


Attributes

Attributes have extra information about an element that is not visible on the displayed page. Some attributes work on several elements and some attributes work on certain elements.

Here are a couple of examples:

  1. Class attribute. These target any element with that class in CSS or JavaScript.

    <p class="Intro"> This is the introduction.</p>

  2. ID attributes. These target specific elements with that ID in CSS or JavaScript. They are similar to class attributes, but can only be used once in an entire HTML page. The unique names are useful for JS and links.

    <p class="Intro" id="Article-Intro"> This is the introduction.</p>

Note that if an element has two or more attributes they need to be spereated by a space.


Boolean attributes

Boolean attributes are attributes without values. As the values 'true' and 'false' are not permitted on boolean attributes, the inclusion of a boolean attributue indicates that the value is true and the absence would mean a false value. The 'value' that they have is the same as the attrribute name or an empty string. For example:

<input type="checkbox" checked="checked">

You can also write it the following way:

<input type="checkbox" checked="">

It is best practise to include the speechmarks for the value to avoid confusion and mistakes with the markup and improve the readability of your code.


Anatomy of an HTML document

There are several key parts that every webpage must contain:

          <!DOCTYPE html>
          <html lang="en-US" dir="ltr">
            <head>
              <meta charset="utf-8">
              <meta name="viewport" content="width=device-width">
              <title>Title Name</title>
            </head>
            <body>
              <h1>This is a headline.</h1>
              <p>1st paragraph.</p>
              <p>Usually there's quite a lot here.</p>
            </body>
          </html>

  1. <!DOCTYPE html>The DOCTYPE html file must begin with a docutype which declares which era this HTML file is from and shows what rules should be followed, this is included so that the HTML works.
  2. <html></html>The HTML element. This is the root element where all other elements will be. The lang attribute refers to the language of the document and the dir attribute refers to the direction of the content written, i.e. for english it would be left to right (ltr) and in other langaues such as arabic it would be right to left(rtl).
  3. <head></head>The head element. This contains all the information the browser needs to know. i.e. the metadata, such as page descriptions which is shown in search results and CSS to style the page's content. This information won't be visible to the users.
  4. <meta charset="utf-8">The meta element. This tells the browser the information about the data in an HTML document e.g. the author, page description and viewport settings (the viewport is the area of the page that the user can see). They are always within the <head> element. The charset attribute refers to the character encoding in the HTML webpage.
  5. <title></title>The title element. This creates the page's title which is visible in the browser tab the webpage is in, as well as the name of the page when it is bookmarked.
  6. <body></body>The body element. This includes all the content that appears in the HTML document i.e. text, videos, images, audio, games etc.

Special characters in HTML

The special characters in HTML are < , > , " , ' and &. In order to refer to them in your code you must use their character reference, which begins with an ampersand (&) and ends with a semicolon (;).

Character Character reference equivalent
< &lt;
> &gt;
" &quot;
' &apos;
& &amp;

HTML comments

To include comments in your code, wrap the content with the special markers: <!-- and -->

For example:

<p> This is not a comment.</p>

<!-- <p>This is!</p> -->

Only the first paragraph will be visible to the user.
